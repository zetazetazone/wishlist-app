## v1.6 Requirements

Requirements for the Wishlist UI Redesign milestone. Each maps to roadmap phases.

### Grid Layout

- [x] **GRID-01**: Wishlist displays as 2-column masonry grid using FlashList
- [x] **GRID-02**: Grid cards show item image with placeholder icon when no URL exists
- [x] **GRID-03**: Grid cards show title (truncated to 2 lines) and price
- [x] **GRID-04**: Grid cards show action button in bottom-right corner (options for owner, claim for others)
- [x] **GRID-05**: Grid layout is consistent between My Wishlist and celebration page views
- [x] **GRID-06**: Special items (Surprise Me, Mystery Box) display with appropriate icons and styling

### Item Detail Page

- [x] **DETAIL-01**: Tapping a grid card opens item detail page via expo-router
- [x] **DETAIL-02**: Detail page shows full-bleed hero image (top to fold)
- [x] **DETAIL-03**: Detail page shows title, brand (parsed from title), and price
- [x] **DETAIL-04**: Detail page shows "Go to Store" button linking to item URL
- [x] **DETAIL-05**: Detail page shows favorite badge and priority stars
- [x] **DETAIL-06**: Detail page has back button and share/options button in header
- [x] **DETAIL-07**: Celebrant sees "Taken" badge when item is claimed (no claimer identity shown)

### Claim UI

- [x] **CLAIM-01**: Claim UI appears on detail page (not grid cards)
- [x] **CLAIM-02**: Group members (except celebrant) can claim items from detail page
- [x] **CLAIM-03**: Claimer can unclaim items from detail page
- [x] **CLAIM-04**: Claimer can open item for split contributions from detail page
- [x] **CLAIM-05**: Split contribution progress displays on detail page for group members
- [x] **CLAIM-06**: Claim state syncs in real-time across views

### Options Sheet

- [x] **OPTIONS-01**: Options sheet opens via action button on grid card (owner) or detail page
- [x] **OPTIONS-02**: Options sheet shows item preview (image, title, price)
- [x] **OPTIONS-03**: Options sheet provides favorite toggle action
- [x] **OPTIONS-04**: Options sheet provides priority (star rating) adjustment
- [x] **OPTIONS-05**: Options sheet provides share action
- [x] **OPTIONS-06**: Options sheet provides edit action (placeholder alert - edit form deferred)
- [x] **OPTIONS-07**: Options sheet provides delete action with confirmation

### Performance

- [x] **PERF-01**: Images load with expo-image caching and blur placeholder
- [x] **PERF-02**: Grid renders smoothly at 60fps during scroll
- [x] **PERF-03**: Detail page loads in <200ms from grid tap

### Feature Parity

- [x] **PARITY-01**: All existing wishlist actions remain functional after redesign
- [x] **PARITY-02**: All existing badge types (favorite, claimed, split) display correctly
- [x] **PARITY-03**: Celebrant privacy rules enforced (sees "taken" not claimer identity)
