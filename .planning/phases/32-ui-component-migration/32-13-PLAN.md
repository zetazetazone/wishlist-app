  <action>
Run ESLint with i18next plugin to detect any remaining hardcoded strings:

1. Run ESLint on all app screens:
   ```bash
   npx eslint "app/**/*.tsx" --rule 'i18next/no-literal-string: warn' 2>&1 | tee /tmp/eslint-app-report.txt
   ```

2. Run ESLint on all components:
   ```bash
   npx eslint "components/**/*.tsx" --rule 'i18next/no-literal-string: warn' 2>&1 | tee /tmp/eslint-components-report.txt
   ```

3. Count remaining warnings:
   ```bash
   grep -c "no-literal-string" /tmp/eslint-app-report.txt || echo "0"
   grep -c "no-literal-string" /tmp/eslint-components-report.txt || echo "0"
   ```

4. Review any remaining warnings:
   - Acceptable exceptions: testID values, technical strings not shown to users
   - Flag any user-facing strings that were missed

5. Verify all useTranslation imports are in place:
   ```bash
   # Count files using useTranslation
   grep -rl "useTranslation" app/ components/ | wc -l
   ```

6. Verify Alert.alert translations:
   ```bash
   # Check for Alert.alert calls that might not use t()
   grep -rn "Alert.alert(" app/ components/ | grep -v "t(" | head -20
   ```
   Any results need review to ensure they're translated.
